# Data Science Methodology Project

This repository contains three data science workflows built around the same project structure:

- Big Five personality analysis
- Crime analysis and modeling
- World Happiness analysis and modeling

It also includes a Streamlit dashboard that exposes the cleaned datasets and the main visual summaries.

## Project Layout

```text
project/
	Dashboards/                Streamlit app and dashboard pages
	Datasets/                  Cleaned CSV outputs used by notebooks and dashboards
	drafts/                    Raw data, working notes, and documentation
	plots/                     Saved figures generated by the notebooks
	crimes_cleaning.ipynb      Crime data cleaning notebook
	crimes_final.ipynb         Crime modeling notebook
	HZ_WH_1Wrangling.ipynb     World Happiness wrangling notebook
	HZ_WH_2Modeling.ipynb      World Happiness modeling notebook
	HZ_WH_helper_functions.py  Helper functions for World Bank data fetching
	oad_big_five_cleaning.ipynb Big Five cleaning notebook
	oad_big_five.ipynb         Big Five modeling and hypothesis testing notebook
	pyproject.toml             Main project dependency definition
	uv.lock                    Locked dependency versions for uv
	requirements.txt           Exported dependency list
```

## Data Folders

- `drafts/datasets/` holds the raw source files used during exploration.
- `Datasets/` holds cleaned CSV files consumed by the notebooks and the dashboard.
- `plots/` stores saved charts, diagnostics, and model visuals.
- `drafts/documentation/` stores supporting notes and report drafts.

## Notebook Guide

- `oad_big_five_cleaning.ipynb`: loads the raw OpenPsychometrics Big Five dataset, removes duplicates and missing rows, applies reverse-keyed scoring, computes trait totals, filters outliers, and writes `Datasets/cleaned_big5_totals.csv`.
- `oad_big_five.ipynb`: loads the cleaned Big Five totals, standardizes the trait columns, produces Q-Q plots and a correlation matrix, and runs regression-based independence checks across the five traits.
- `crimes_cleaning.ipynb`: cleans the crime dataset and writes `Datasets/cleaned_crime_data.csv` for downstream analysis.
- `crimes_final.ipynb`: performs the crime analysis and modeling workflow, compares classifiers, evaluates performance, and exports model artifacts and plots.
- `HZ_WH_1Wrangling.ipynb`: fetches and wrangles World Bank indicators, merges them with the World Happiness data, and produces the analysis-ready dataset used later in the workflow.
- `HZ_WH_2Modeling.ipynb`: trains and evaluates the World Happiness models, compares alternatives, and saves model outputs and diagnostic plots.
- `HZ_WH_helper_functions.py`: helper script for World Bank fetching and missing-data reporting used by the World Happiness workflow.

## Dashboard

The Streamlit app lives in `Dashboards/` and is started from `Dashboards/dashboard.py`. It includes pages for:

- Big Five analysis
- Crime analysis
- World Happiness analysis
- a general-purpose data cleaning and preprocessing tool

Run it from the `Dashboards/` folder so its relative dataset paths resolve correctly.

## Setup

The project is managed with `uv`, but you can also set it up with standard Python and `pip`.

### Option 1: uv

1. Install `uv` if it is not already available.
2. From the project root, create or refresh the environment:

```bash
uv sync
```

3. Open the notebooks in VS Code using the project interpreter, or start a notebook server if you prefer.

### Option 2: Python and pip

1. Install Python 3.14 or newer.
2. Create and activate a virtual environment from the project root:

```bash
python -m venv .venv
source .venv/bin/activate
```

3. Install the dependencies with pip:

```bash
pip install -r requirements.txt
```

4. Open the notebooks in VS Code using the `.venv` interpreter, or start a notebook server if you prefer.

## Run the Dashboard

From the `Dashboards/` directory:

```bash
uv run streamlit run dashboard.py
```

That starts the multipage Streamlit app with the dashboard pages listed above.

If you are using `pip` instead of `uv`, activate the virtual environment first and then run the same command from `Dashboards/`:

```bash
source ../.venv/bin/activate
streamlit run dashboard.py
```

## Run the Notebooks

- Open the notebook you want to inspect or rerun in VS Code.
- Use the project environment created by `uv sync`.
- Re-run the notebooks in order if you need to regenerate cleaned CSV files or plots.

Suggested notebook order:

1. `oad_big_five_cleaning.ipynb`
2. `oad_big_five.ipynb`
3. `crimes_cleaning.ipynb`
4. `crimes_final.ipynb`
5. `HZ_WH_1Wrangling.ipynb`
6. `HZ_WH_2Modeling.ipynb`

## Notes

- `requirements.txt` is an exported dependency snapshot; `pyproject.toml` and `uv.lock` are the primary dependency files.
- The project writes cleaned data and plots into the repository, so rerunning notebooks may update files under `Datasets/` and `plots/`.
- The `.venv/` folder is local environment state and should not be edited manually.
